UFD base demonstration page

basic combo

Try this basic example usecase for UFD. It works like a normal dropdown, but you can start typing to filter.


the master select tag: inside UFD

This is the same basic dropdown; it wraps the hidden (original) master select node. Events on the master are logged on the area to the right; so this log is from the events of the ORIGINAL master select box firing.

This means that the UFD widget can be initialized and destroyed with the events remaining in place, events can be bound and removed idependant to UFD. UFD triggers some but not all, events on the master: namely the focus, change and blur events.


Performance test

This demonstrates the initialisation time. Type in a number of items to generate, and click the button.

Timing is split 2: string generation + option building ; and the actual initialization given the existing <select node. The latter is the relevant part, as the first is just generating dummy data!

You can put any number, but keep it sane (<: 10,000) as initialization is proportional to list length.




width control

Sizing defaults to autosize. UFD sets the exact same width as the master select, unless the original is below the "minWidth" option, or manualWidth is set.










Free text

The combobox below allows freetext, submitting the text back using the original selects' name. All events are still propagated back to master (including attempt to set if matched) even though it has been renamed (suffixed).


Functions

This shows the useful functions you can call on widgets. There are duplicate labels, so you can't filter to a single value. The original order is preserved, and you can create and destroy multiple times to confirm that the correct instance is selected in each case.







Skins and functions

the 3 skins bundled with UFD. The last one is a CSS only template to help you markup your own needs.







zIndex correction on iE6

The combobox works over select input, working around the iE6 zindex problem using bgiframe.
